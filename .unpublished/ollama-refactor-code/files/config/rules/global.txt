## SCOPE OF REVIEW (STRICT)

**CRITICAL: CODE FORMATTING IS NOT YOUR RESPONSIBILITY.**

Your analysis **MUST IGNORE** all purely stylistic and formatting aspects of the code. This includes:
- Line length
- Whitespace (indentation, trailing spaces, blank lines)
- Wrapping of long lines (function signatures, lists, etc.)

**DO NOT** suggest changes that only alter formatting. If the code is functionally and architecturally sound but does not match your personal formatting style, you MUST consider it **CORRECT**.

Your focus is exclusively on:
1.  **Logic Flaws & Bugs**
2.  **Security Vulnerabilities**
3.  **Architectural Violations (SOLID, DRY)**
4.  **Type Safety & Correctness**
5.  **Possible Optimizations**
6.  **Language-Specific Best Practices**
